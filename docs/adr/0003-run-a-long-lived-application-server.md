# Run a long-lived application server

Version one runs as a single long-lived application container with persistent real-time client connections; purely serverless deployment is intentionally excluded. The repository ships one production image and a Docker Compose setup with separate mounts for live SQLite data and rotating backups, while TLS and the public domain remain the hosting reverse proxy's responsibility. A separate mount protects against database-volume failure only when the operator maps it to independent or off-host storage; the default topology does not by itself guarantee host-loss recovery. The expected audience of fewer than 100 concurrent participants, with 500 as a stretch goal, favors simple and predictable live synchronization over serverless scaling and connection constraints.
